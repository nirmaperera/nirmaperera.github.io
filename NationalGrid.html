National Grid Login 2.0

A Redesign of National Grid login's website

Tech Stack

  • React
  • Redux
  • SASS
  • View Code
  • View Wireframes

My Role

Frontend Application

I wanted to redesign the currect login and register page for National Grid, because I believed that those pages were outdated. My inspiration for the design was the ConEdison website. In addition, I tried my best to match the styles and colors of the existing of national grid wesbite. In the proces of doing so, I wanted to familirse with flexbox and SASS. However, during the process I eventually ended up adding more logic like user authenticaton, state persistance, etc.

Project Difficulties

Some of the Difficulties I faced was state managemnt, private routes, and state persistence. I wanted a way to check if the user was logged in, and if so, change the Navigation bar, and vice versa. Simarlarly, I wanted to prevent the user from accessing dashboard and profile route if the user was not logged in. Lastly, because the redux store gets return to its inital state upon page refresh, i wanted a wata to persist the state.

Project Solutions

One way I managed state was by using Redux. Although there was alot of boilerplate code, I found that it was easy to grasp and good to familarize with. In case, the project was to scale, using redux is beneficial. I solved the private route issue, by creating a PrivateRoute function and passing isLogged(state in redux) as a prop to PrivateRoute. Finally, I persisted redux state to local storage by writing two function, one which saves the state to local storage, and the other which load the state from local storage.

Application Features

  • Registration
  • Login
  • User authentication
  • Profile (change user info)
  • Dashboard (Pay Bill)

What's Missing?

The Application only contains Login, Register, Dashboard, and Profile components. It can scale to other conponents like View bill, stop and start service, etc, given the resources.
