Wine Selector

Find your perfect wine with only a few questions.

Tech Stack

  • React
  • Python
  • Django
  • PostgreSQL
  • Quini Wine API
  • AWS EC2
  • View Code
  • View Wireframes

My Role

Frontend developer and designer

The goal of the application was to recommend wines based on user’s preference, provide a clean web user interface to navigate, display information clearly, and store user's information.

Project Difficulties

Most of the difficulties from the front end was design based, which included making the application responsive. In addition, learning React for the first time was challenging. As a group some difficulties we faced was agreeing on the stack, and deciding on recommendation algorithm

Project Solutions

Developer tools helped me tremendously when it came to fixing design layout bugs. When it came to react, due to the limited time, I decided to create react components instead using the create-react-app. For example, I made components from react, such as the navbar and buttons, which were components that were reusable on multiple pages. In addition, after some discussions around how to choose a recommendation algorithm, we decided to use a simple filtering method which filtered data based on user preferences (wine type, region, and vintage), so that we can reduce the risk of delivering a incomplete product.

Application Features

  • Registration
  • Login
  • Profile
  • take a quiz, to find wines that match your perference

What's missing?

  • Recommendation Algorithm- We could have implemented more complicated recommendation algorithm like content-based or clustering method.
  • Recommendation History- A user can store their history of wine recommendations, so that the user can access these wines later on.
  • Reviews- A user can review each recommend wine for her/him, so it generates a feedback loop to the database, ultimately refining the recommendation algorithm.
